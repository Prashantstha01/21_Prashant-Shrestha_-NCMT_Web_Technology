# CHAPTER 2: WEBSITE DEVELOPMENT

## Group A: Short Questions (2 Marks)

### 1. Define Information Architecture (IA).
**Ans.** The process of arranging and structuring data in a logical and understandable manner so that it is simple to locate, comprehend, and utilize is known as Information Architecture (IA).

### 2. What is a “Wireframe”?
**Ans.** A wireframe is a straightforward visual outline or sketch of a webpage or app. It shows where important elements like text, images, and buttons will be placed.

### 3. Explain the concept of “Cognitive Friction” in web design.
**Ans.** Cognitive friction in web design refers to anything on a website that makes users stop and think too hard. This includes confusing layouts or unclear instructions.

### 4. Why should URL slugs use hyphens instead of underscores?
**Ans.** URL slugs should use hyphens instead of underscores because hyphens are easier for people and search engines to read, as they clearly separate words, while underscores can make words look joined together and harder to understand.
 
 # Group B: Long Questions (4 Marks)

## 5. “Planning is the most critical phase of web development.” — Discuss

Planning is the foundation of the entire web development process. If this phase is weak, the website may fail even if coding is done properly.

### Key activities during Planning:

- Defining the purpose and goals of the website  
- Identifying the target audience  
- Collecting functional requirements  
- Creating sitemap and page structure  
- Selecting tools and technologies  
- Estimating budget and project timeline  

---

## 6. Iterative nature of the Maintenance Phase  
### Why a website is never truly “finished”

The Maintenance phase is continuous and repetitive. Even after launch, the website must be updated regularly.

### Reasons:

- New content is added frequently  
- Bugs and errors appear over time  
- User feedback requires changes  
- Security threats keep increasing  
- Technology and design trends change  

### Maintenance activities:

- Updating content  
- Fixing bugs  
- Improving speed and performance  
- Adding new features  
- Strengthening security  

---

## 7. Information Gathering Process  
### Importance of understanding the target audience

Information gathering is the first step where all project details are collected.

### Activities involved:

- Client meetings  
- Identifying goals and objectives  
- Analyzing competitors  
- Listing features  
- Understanding user needs  

### Why target audience matters:

If the audience is not understood, the website may become confusing, unattractive, and ineffective.

# Group C: Scenario-Based Questions (5 Marks)

## 8. Risks of skipping the Testing phase

Skipping testing may lead to serious problems:

| Area | Risk |
|------|------|
| Security | Customer data may be hacked |
| Functionality | Payment or login system may fail |
| Compatibility | Website may not work on all devices |

**Conclusion:** Testing saves money in the long run and protects user trust.

## 9. Developer coded without design approval — What went wrong?

### Mistakes:

- No prototype was prepared  
- No sitemap was created  
- Client feedback was ignored  

### Correct approach:

- Create wireframes  
- Design layout first  
- Take client approval before coding
